# SORGENTE UNICA dei vincoli runtime: `requirements.in`. Questo file la richiama con
# `-r` così c'è UN solo posto dove aggiungere/cambiare un dipendenza runtime, e quel

# generato su Windows). Niente più drift tra requirements.txt e il lock: una nuova
# dipendenza runtime DEVE entrare in `requirements.in` (vedi README, "Dependency
# lockfile / build EXE riproducibile").
-r requirements.in
